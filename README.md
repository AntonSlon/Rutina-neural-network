# Rutina Neural Network

`Rutina Neural Network` — NLP-сервис для классификации пользовательских запросов и подбора релевантного совета из подготовленной базы.

Проект сочетает:
- `RuBERT` для классификации запроса по крупной категории
- `SentenceTransformer` для dense retrieval внутри выбранной категории
- `BM25` + `pymorphy3` для lexical reranking на русском языке
- `CrossEncoder` для дополнительного reranking длинных запросов
- `FastAPI` для API-слоя

## Что делает модель

Система работает по гибридному retrieval pipeline:

1. Пользователь отправляет текстовый запрос.
2. Запрос векторизуется через `RuBERT`.
3. Классификатор предсказывает одну из трёх категорий:
   - `0` — питание
   - `1` — спорт
   - `2` — учёба / продуктивность
4. Из базы советов берутся только тексты внутри предсказанной категории.
5. Внутри категории выполняется dense retrieval:
   - запрос кодируется через `SentenceTransformer`
   - советы представлены заранее сохранёнными sentence embeddings
   - top-кандидаты отбираются по `cosine similarity`
6. Для кандидатов выполняется lexical reranking через `BM25`.
7. Перед `BM25` тексты и запрос лемматизируются через `pymorphy3`, чтобы лучше обрабатывать русские словоформы.
8. Для коротких запросов финальный совет выбирается по `BM25`.
9. Для длинных запросов top кандидаты после `BM25` дополнительно переранжируются через `CrossEncoder`.
10. Пользователю возвращается лучший совет.

Кратко:

`запрос -> RuBERT -> классификатор -> категория -> dense retrieval -> BM25 -> optional CrossEncoder -> совет`

## Почему именно hybrid retrieval

Изначально retrieval внутри категории строился только на semantic similarity. На коротких запросах вроде `бег`, `турник`, `планка` этого оказалось недостаточно: система попадала в правильную категорию, но иногда выбирала слишком общий или тематически смещённый совет.

Текущий pipeline решает это так:
- dense retrieval сохраняет семантическую близость
- `BM25` усиливает точные ключевые слова
- `pymorphy3` помогает сопоставлять русские словоформы
- `CrossEncoder` применяется только там, где запрос достаточно длинный и несёт больше контекста

## Структура проекта

- `main.py` — точка входа FastAPI-сервиса
- `src/ruBERT.py` — работа с `DeepPavlov/rubert-base-cased`
- `src/classifier.py` — архитектура классификатора
- `src/train.py` — обучение классификационной модели
- `src/predict.py` — inference pipeline: классификация, dense retrieval, BM25 и optional reranking
- `src/api/schemas/` — схемы запроса и ответа
- `data/` — датасеты, eval-примеры и база советов
- `model/` — обученные артефакты и сериализованный retrieval-индекс
- `reports/metrics/` — скрипты оценки качества
- `reports/grids/` — скрипты построения графиков


Эти метрики относятся именно к задаче классификации категории. Качество финального совета дополнительно зависит от retrieval-базы и hybrid retrieval pipeline.

## API

По умолчанию сервис запускается на `http://0.0.0.0:8000`.

### `GET /`

Проверка работоспособности сервиса.

Ответ:

```json
{
  "message": "this is rutina neural network service"
}
```

### `POST /advice`

Основной endpoint для подбора совета.

Тело запроса:

```json
{
  "habit": "Бег 400м 5 подходов с максимальным восстановлением"
}
```

Ответ:

```json
{
  "advice": "Текст подобранной рекомендации"
}
```

## Как запустить

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Пересобрать индекс советов

```bash
python -m src.predict
```

Эта команда пересоздаёт `model/tips_ru.pt` на основе `data/tips.json`.

### 3. Запустить API

```bash
python main.py
```

## Важные замечания

- Высокая `category accuracy` не гарантирует идеальный финальный совет.
- Основная сложность задачи находится не только в классификации, но и в retrieval внутри широкой категории.
- Качество `tips.json` напрямую влияет на итоговый ответ модели.
- Hybrid retrieval особенно полезен для коротких и конкретных запросов.
- `CrossEncoder` не всегда улучшает ответ: для коротких запросов лучший результат часто даёт связка `dense retrieval + BM25`.

## Возможные улучшения

- считать retrieval-метрики вроде `precision@k`, `recall@k`, `MRR`, `nDCG`
- расширить базу советов и сделать её более равномерной внутри категорий
- разбить широкие классы вроде спорта на более узкие подтемы
